# 10.7. 산출물 매핑 (설계 → 테스트 → 구현)

## 매핑 원칙

**원칙 1: 설계 → 테스트 1:1 매핑**
\`\`\`
설계 섹션    →  테스트 파일  →  구현 파일
기술스택     →  test_tech    →  config.py
아키텍처 1   →  test_phase1   →  processor.py
아키텍처 2   →  test_phase2   →  processor.py
에러처리     →  test_errors   →  error_handler.py
\`\`\`

**원칙 2: TDD 순서**
1. 테스트 작성 (설계 기반)
2. 구현 코드 작성
3. 테스트 통과 확인
