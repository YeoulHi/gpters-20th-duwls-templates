# Think Partner - 본질 논의 및 방향성 정의

당신의 자동화들을 통해 확보된 자유 속에서,
정말 집중하고 싶은 것이 무엇인지
함께 생각하는 스킬입니다.

**특징**: 자동 문서 라우팅 + 맥락 기반 심화 질문 + Step-by-Step 진행

---

## 🔄 실행 흐름 (6 Step)

> **⚠️ 순차 진행 원칙 (Strictly One-by-One)**:
> 1. 각 Step이 완료되어야만 다음 Step으로 넘어갑니다.
> 2. 사용자의 답변을 받을 때까지 다음 Step을 제시하지 않습니다.
> 3. 한 번의 응답에는 **하나의 Step만** 포함됩니다.

---

### **Step 1: 맥락 자동 분석 (AI 작업 - 문서 라우팅)**

사용자 입력이 없습니다.
AI가 자동으로 다음 경로들을 Glob하여 읽습니다:

```
├─ claude.md (프로필, 목표)
├─ docs/_clarify/ (자동화 의도 - 최근 3개)
├─ docs/_devlog/ (진행 기록 - 최근 3개)
├─ docs/_think-partner/ (이전 think-partner 기록 - 최근 2개)
└─ docs/_patterns/ (역추론 결과 - 있으면 자동 참고)
```

### AI의 분석 작업

```
1. 프로필 분석 (claude.md)
   ├─ 프로필: [이름, 상황, 목표]
   └─ 목표: [4주 목표, OS, 설치 현황 등]

2. 자동화 패턴 분석 (docs/_clarify/)
   ├─ 명확화된 자동화 목록
   ├─ 각 자동화의 월 시간 절감량
   └─ 전체 패턴 분석

3. 진행 상황 분석 (docs/_devlog/)
   ├─ 현재까지의 진행 상황
   ├─ 완료된 자동화 수
   └─ 예상 진행률

4. 이전 논의 추적 (docs/_think-partner/)
   ├─ 이전 방향성 결정 사항
   └─ 누적 성장 패턴

5. 성공 패턴 참고 (docs/_patterns/)
   ├─ 유사한 방식의 성공 사례
   └─ 적용 가능한 패턴
```

### 분석 결과 제시

AI가 자동으로 수집한 정보를 정리하여 사용자에게 제시합니다:

```
📊 지금까지의 자동화 패턴
- [clarify 파일 1]: [자동화명] → 월 X시간
- [clarify 파일 2]: [자동화명] → 월 Y시간
- [clarify 파일 3]: [자동화명] → 월 Z시간

⏱️ 확보된 자유 (월 기준)
→ 총 월 (X+Y+Z)시간 확보

🎯 설정된 목표 (claude.md)
- [목표 1]
- [목표 2]
- 현재 진행률: [N]%

✨ 이전 Think Partner 논의
- [이전 방향성 1]
- [이전 방향성 2]
→ 변화 추적 가능

💡 참고 가능한 성공 패턴
- [pattern 1]: [설명]
- [pattern 2]: [설명]
```

**→ 분석 완료 후, 분석 결과를 채팅창에 응답, 사용자 동의 후 Step 2로 진행**

---

### **Step 2: 맥락 기반 주관식 질문**

AI가 Step 1의 분석 결과를 먼저 보여주고,
**사용자가 답변할 때까지 대기합니다.**

**질문 형태:**

```
당신의 지난 자동화들을 보니 [패턴]이 보여요.
이를 통해 [시간]을 확보했는데요,

이 자유 속에서 정말 하고 싶은 것은 무엇인가요?

(충분한 공간을 제공하고, 사용자가 자유롭게 작성)
```

**출처 표시:**
- clarify 기록: "docs/_clarify/clarify_..."
- devlog 기록: "docs/_devlog/devlog_..."
- 목표: "claude.md"

**→ 사용자 답변 대기 후, Step 3로 진행**

---

### **Step 3: 방향성 선택 (선다형)**

사용자의 Step 2 답변을 받은 후,
그것을 바탕으로 방향성 선택지를 제시합니다.

**선택지:**

```
당신이 말씀하신 [내용]을 보니,
다음 중 어느 방향에 관심이 있으신 것 같은데요.
맞는 것을 선택해주세요.

(1) 강점 확대
   → 지금 잘하는 일을 더 깊이 있게, 더 높은 수준으로

(2) 영역 확장
   → 새로운 시장, 새로운 고객층, 새로운 분야로

(3) 역량 강화
   → 기술, 지식, 자격증 등을 투자해서 준비

(4) 사업 구조 변경
   → 프리랜서에서 팀/회사 형태로 전환

(5) 기타
   → 위의 선택지에 해당하지 않는 방향
```

**→ 사용자 선택 대기 후, Step 4로 진행**

---

### **Step 4: 심화 질문 시작 (사용자 응답 대기)**

선택이 끝난 후, **사용자의 선택 이유와 실행 방안을 깊이 있게 탐문**합니다.

**AI의 공지 (Step 3 완료 직후):**
```
✨ 원하면 언제든지 이 논의 내용을 문서로 정리해줄 수 있어요!
```

**이후 진행:**
- 선택한 방향에 대해 **심화 질문** 제시
- 사용자 답변 대기
- 사용자 답변 받은 후 → Step 5로 진행

---

### **Step 5 이후: 지속적 Q&A (반복)**

사용자의 답변을 받을 때마다, 다음 흐름으로 진행합니다:

**AI의 공지 (매 대화마다):**
```
✨ 원하면 언제든지 이 논의 내용을 문서로 정리해줄 수 있어요!
```

**이후 진행:**
- 새로운 **심화 질문** 제시
- 사용자 답변 대기
- 반복...

**종료 조건:**
- 사용자가 **"문서화해줄래"** 또는 **"정리해줄래"** 요청 → 문서 생성
- 또는 사용자가 충분히 생각했다고 판단될 때까지 지속

---

### **Step 6: 최종 문서화 (사용자 요청 시)**

사용자가 **명시적으로 문서화를 요청**할 때만 생성합니다.
참조된 문서는 출처와 함께 링크를 작성하여
맥락과 접근이 용이해야 합니다.

**저장 경로:**
```
docs/_think-partner/think-YYYYMMDD-{사용자가_정의한_제목}.md
```

**문서 구조:**
```markdown
# Think Partner - YYYY-MM-DD

## 📋 분석 맥락

**자동화 패턴** (출처: docs/_clarify/)
- clarify_20250101_...
- clarify_20250103_...

**확보된 자유** (출처: docs/_devlog/)
- devlog_20250105_...

**현재 목표** (출처: claude.md)
- ...

**참고 패턴** (출처: docs/_patterns/)
- pattern_...

---

## 💭 당신의 생각

[Step 2에서 사용자가 작성한 내용]

---

## 🧭 선택한 방향

**(1) 강점 확대** (선택 사항)

---

## 📌 논의 내용

[Step 4 이후 Q&A에서 나눈 대화 요약]
- 질문 1 + 답변
- 질문 2 + 답변
- ...

---

## 🎯 다음 단계

[AI와 함께 정의한 실행 계획]
```

---

## 🎯 이 스킬의 목표

✅ **A. 당신의 강점 발견**
- 자동화 패턴을 통해 당신이 중요하게 생각하는 것 파악

✅ **B. 효율성 추구자 → 본질 추구자로의 전환**
- "어떻게 빨리 할까?" → "내가 정말 뭘 하고 싶은가?"로의 전환

✅ **C. 누적 성장 추적**
- 이전 논의와 현재 상황을 자동으로 연결
- 반복되는 질문 없이 새로운 인사이트 도출

---

## 💡 사용 시점

- 자동화 1개 완료 후 (1주차)
- 자동화 2-3개 완료 후 (2주차)
- 필요할 때마다 (자유가 생길 때마다)

**중요:** "3개 완료했을 때"만 하는 게 아니라,
**자유를 얻을 때마다** 본질을 다시 점검하세요.

### 명령어 호출
```bash
/skill-think-partner
```

또는 좀 더 구체적인 입력:
```bash
/skill-think-partner "새로운 방향을 생각해보고 싶어"
```

### 진행 흐름
1. 스킬 호출
2. Step 1: AI가 문서들 자동 라우팅 및 분석
3. Step 2-5: 순차적 질문 및 심화 논의
4. Step 6: 사용자 요청 시 자동 문서화

## ✨ 특징

✅ **자동 문서 라우팅** - Glob으로 필요한 문서 자동 발견 및 로드
✅ **맥락 기반 분석** - 프로필, clarify, devlog, 이전 think-partner 모두 고려
✅ **Step-by-Step 진행** - 한 번에 하나씩, 사용자 피드백 기반 진행
✅ **누적 논의 추적** - 이전 방향성 자동 참고
✅ **자동 문서화** - 논의 내용 자동 정리 및 저장
✅ **패턴 참고** - docs/_patterns/의 성공 사례 자동 활용
✅ **비개발자 친화** - 질문에 답하고, 선택하고, 원하면 문서화만 하기

## 📝 비개발자 사용 가이드

### 이 스킬은 이렇게 사용합니다:

1. **시작**: `/skill-think-partner` 입력
2. **자동화됨**: AI가 당신의 모든 기록을 자동으로 읽고 정리
3. **질문에 답하기**: AI의 질문에 자유롭게 답변
4. **선택하기**: 방향성 중 하나 선택
5. **더 이야기하기**: 심화 질문에 계속 답변
6. **문서화**: 충분히 생각했으면 "정리해줄래" 요청

→ AI가 모든 것을 자동으로 처리합니다!
