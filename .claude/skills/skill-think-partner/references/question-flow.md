# 질문 흐름 가이드 (Question Flow)

Think Partner Skill의 Step 2부터 Step 5까지 실행되는 질문의 흐름과 가이드입니다.

---

## 📋 Step 2: 맥락 기반 주관식 질문

### 목표
사용자의 자유로운 생각을 이끌어내기

### 질문 형태

```
당신의 지난 자동화들을 보니 [패턴]이 보여요.
이를 통해 [시간]을 확보했는데요,

이 자유 속에서 정말 하고 싶은 것은 무엇인가요?

(충분한 공간을 제공하고, 사용자가 자유롭게 작성)
```

### 패턴 해석 방법

#### 패턴 감지 예시:

```
clarify 파일들을 분석하면:
├─ SNS 관리 자동화
├─ 데이터 분류 자동화
├─ 반복 보고서 자동화
→ 패턴: "데이터 처리에 주력하고 있다"

또는:
├─ 업체 섭외 자동화
├─ 계약서 정리 자동화
├─ 송장 관리 자동화
→ 패턴: "사무 업무 효율화에 집중하고 있다"

또는:
├─ 영상 편집 자동화
├─ 썸네일 생성 자동화
├─ 자막 처리 자동화
→ 패턴: "크리에이티브 작업 자동화에 주력하고 있다"
```

### 시간 계산

```
clarify 파일에서 추출:
- Task 1: 월 10시간
- Task 2: 월 8시간
- Task 3: 월 5시간

표현: "월 23시간을 확보했는데요"
또는: "연간 약 276시간을 절감했는데요"
```

### 응답 해석 가이드

#### 사용자가 이렇게 답하면:

```
"더 많은 프로젝트를 할 수 있을 것 같아요"
→ 해석: 영역 확장 관심 (더 큰 일감 원함)

"지금 하는 일을 더 잘하고 싶어요"
→ 해석: 강점 확대 관심 (깊이 추구)

"새로운 기술을 배워보고 싶어요"
→ 해석: 역량 강화 관심 (학습 욕구)

"팀을 꾸려서 더 큰 사업을 하고 싶어요"
→ 해석: 사업 구조 변경 관심 (스케일업)

"뭘 해야 할지 모르겠어요"
→ 해석: 명확화 필요 (추가 질문 필요)
```

## 🎯 Step 3: 방향성 선택 (선다형)

### 프로세스

```
1. Step 2의 사용자 답변 분석
2. 가장 적합한 방향 제시 (1순위)
3. 다른 가능한 방향들도 함께 제시
4. 사용자 선택 대기
```

### 선택지 제시 순서

```
당신이 말씀하신 "[사용자의 답변 요약]"을 보니,
다음 중 어느 방향에 관심이 있으신 것 같은데요.
맞는 것을 선택해주세요.

(1) 강점 확대 ← 가장 적합한 선택지
(2) 영역 확장
(3) 역량 강화
(4) 사업 구조 변경
(5) 기타
```

---

## 💭 Step 4+: 심화 질문 (반복)

### 목표
선택한 방향을 구체적으로 실행 가능한 계획으로 전환

### 일반 심화 질문 (모든 방향에 공통)

```
Q1: "[방향]을 선택하신 이유가 뭘까요?"
목적: 선택의 확신도 확인

Q2: "그렇게 되면, 당신의 삶에서 뭐가 바뀔까요?"
목적: 구체적 미래상 그리기

Q3: "지금부터 할 수 있는 첫 단계는 뭘까요?"
목적: 실행 계획 수립

Q4: "그 과정에서 가장 어려울 것 같은 게 뭐죠?"
목적: 리스크 파악

Q5: "그 어려움을 어떻게 극복하실 생각이신가요?"
목적: 대응 방안 도출
```

---

## 🎯 방향별 심화 질문 (상세)

### (1) 강점 확대 - 심화 질문

```
Q1: "지금 당신이 가장 강하다고 생각하는 분야가 뭔가요?"
    → 세부 예: "영상 편집 중에서도 어떤 분야?"
    → 답변 예: "자막 처리와 색감 보정"

Q2: "그 분야에서 당신만의 차별점이 뭘까요?"
    → 세부 예: "다른 편집자와 다른 점?"
    → 답변 예: "빠른 턴어라운드, 트렌디한 스타일"

Q3: "이 강점을 더 발전시키려면 어떤 기술이나 지식이 필요할까요?"
    → 세부 예: "어떤 교육이나 경험?"
    → 답변 예: "AI 자동 자막 기술, 고급 컬러 그레이딩"

Q4: "강점이 깊어지면, 그것으로 뭘 할 수 있을까요?"
    → 세부 예: "어떤 고객을 대상으로?"
    → 답변 예: "대형 유튜브 채널, 광고사 영상"

Q5: "이 방향을 위해 당신이 할 수 있는 첫 단계는?"
    → 세부 예: "언제부터, 누구에게?"
    → 답변 예: "이번 주 유튜브 튜토리얼 학습"

Q6: "예상되는 성과는 뭘까요?"
    → 답변 예: "시간당 단가 상승, 더 큰 프로젝트 수주"
```

### (2) 영역 확장 - 심화 질문

```
Q1: "당신의 기술/경험을 필요로 하는 다른 분야가 있을까요?"
    → 세부 예: "현재 영상 편집 외에?"
    → 답변 예: "팟캐스트, 라이브 스트리밍"

Q2: "새로운 영역에는 어떤 고객들이 있을까요?"
    → 세부 예: "그들의 특징?"
    → 답변 예: "개인 크리에이터, 소상공인"

Q3: "그들의 가장 큰 pain point는 뭘까요?"
    → 세부 예: "무엇 때문에 어려워하나?"
    → 답변 예: "편집할 시간이 없음, 비용 부담"

Q4: "당신이 해결해줄 수 있는 게 뭘까요?"
    → 세부 예: "구체적인 서비스?"
    → 답변 예: "팟캐스트 편집 + 유튜브 영상화 패키지"

Q5: "첫 고객 확보를 위해 할 수 있는 게?"
    → 세부 예: "구체적인 액션?"
    → 답변 예: "팟캐스트 커뮤니티에 포트폴리오 공유"

Q6: "그 영역에서 예상되는 기회는?"
    → 답변 예: "월 3-4개 추가 프로젝트, 월 수입 50% 증대"
```

### (3) 역량 강화 - 심화 질문

```
Q1: "앞으로 가장 필요한 역량이 뭐라고 생각하세요?"
    → 세부 예: "부족하다고 느끼는 게?"
    → 답변 예: "마케팅 기초, 영업 스킬"

Q2: "그 역량을 어디서/어떻게 배울 수 있을까요?"
    → 세부 예: "어떤 과정, 책, 멘토?"
    → 답변 예: "온라인 마케팅 강의, 마케팅 책 읽기"

Q3: "배우는 데 얼마나 걸릴 것 같은가?"
    → 세부 예: "주단위, 월단위로?"
    → 답변 예: "3개월 정도 집중이면 기초는 다질 수 있을 것 같아요"

Q4: "배운 후에는 뭘 할 수 있을까요?"
    → 세부 예: "어떤 변화가 생길까?"
    → 답변 예: "직접 마케팅까지 할 수 있는 완전한 서비스 제공"

Q5: "첫 학습 단계로 할 수 있는 게?"
    → 세부 예: "이번 주 또는 이번 달?"
    → 답변 예: "마케팅 기본 강의 1개 수강"

Q6: "이 역량이 생기면, 어떤 기회가 생길까요?"
    → 답변 예: "더 큰 고객 확보, 자신의 브랜드 구축 가능"
```

### (4) 사업 구조 변경 - 심화 질문

```
Q1: "팀을 꾸린다면, 누가 필요할까요?"
    → 세부 예: "어떤 역할, 몇 명?"
    → 답변 예: "영상 편집자 1명, 마케터 1명"

Q2: "그들과 어떻게 일하고 싶으세요?"
    → 세부 예: "고용? 협력사? 파트너?"
    → 답변 예: "처음엔 협력사로 시작, 나중에 정직원"

Q3: "사업 구조 변경 시 가장 큰 과제는?"
    → 세부 예: "리스크? 비용? 관리?"
    → 답변 예: "초기 자본 확보, 팀원 관리"

Q4: "그 과제를 어떻게 극복할 수 있을까요?"
    → 세부 예: "구체적인 방법?"
    → 답변 예: "수익의 일부를 재투자, 경영 스쿨 다니기"

Q5: "첫 팀원이나 파트너는 누가 될 수 있을까요?"
    → 세부 예: "알고 있는 사람? 채용?"
    → 답변 예: "현재 협력 중인 편집자"

Q6: "팀이 구성되면, 당신의 역할은 뭐가 될까요?"
    → 답변 예: "전략 및 영업, 품질 관리"
```

### (5) 기타 - 심화 질문

```
Q1: "당신의 정말 궁극의 목표는 뭘까요?"
    → 세부 예: "돈? 명성? 영향력? 자유?"
    → 답변 예: "지속 가능한 라이프스타일 구축"

Q2: "그 목표를 달성하려면 뭐가 필요한가?"
    → 세부 예: "조건, 환경, 역량?"
    → 답변 예: "월 생활비 200만원 이상, 자유로운 일정"

Q3: "현재와 목표 사이의 간격은?"
    → 세부 예: "무엇이 부족한가?"
    → 답변 예: "월 수입은 충분하지만, 시간 자유도가 낮음"

Q4: "그 간격을 줄이기 위한 첫 단계는?"
    → 세부 예: "무엇부터 시작할까?"
    → 답변 예: "고정 클라이언트 확보로 일정 예측 가능하게"

Q5: "이 방향이 정말 맞는 것 같으세요?"
    → 세부 예: "확신도는?"
    → 답변 예: "네, 이게 제가 진정 원하는 것 같아요"

Q6: "이 방향으로 가기 위해 구체적으로 뭘 하실 거예요?"
    → 답변 예: "고정 클라이언트 3곳 확보 목표"
```

## 🔄 질문 진행 패턴

### 기본 패턴

```
1차 질문:
├─ 개방형 질문 (사용자의 생각을 이끌어냄)
└─ 예: "이 강점을 더 발전시키려면 뭐가 필요할까요?"

사용자 답변 대기

2차 질문 (조정):
├─ 사용자의 답변을 반영한 구체화
└─ 예: "그렇다면 구체적으로 언제부터 시작하실 거예요?"

사용자 답변 대기

3차 질문 (심화):
├─ 리스크, 기회, 다음 단계 등 탐문
└─ 예: "그 과정에서 가장 어려울 것 같은 게 뭐죠?"

...계속 반복
```

### 답변 해석 가이드

#### 사용자가 구체적으로 답할 때:
```
좋은 신호 ✅
"Python을 배워서 자동화 복잡도를 올리고 싶어요"
→ 다음 질문: "그럼 언제부터 어떻게 배우실 거예요?"
```

#### 사용자가 모호하게 답할 때:
```
추가 질문 필요 ⚠️
"더 큰 일을 하고 싶어요"
→ 다음 질문: "큰 일이라고 하면, 구체적으로 어떤 일을 말씀하는 건가요?"
```

#### 사용자가 불안감을 표할 때:
```
공감 + 현실화 🤝
"할 수 있을까 모르겠어요"
→ 다음 질문: "그 불안감의 근거가 뭘까요? 구체적으로?"
```

## 📊 Step별 질문 구조 요약

| Step | 질문 유형 | 개수 | 목적 | 답변 형식 |
|------|---------|------|------|---------|
| 2 | 개방형 | 1개 | 생각 유도 | 자유 작성 |
| 3 | 선택형 | 1개 | 방향 결정 | 선택지 선택 |
| 4+ | 심화형 | 5-6개 | 구체화 & 실행 계획 | 자유 답변 |

---

## ✨ 질문 시 주의사항

### DO ✅

```
1. 한 번에 한 질문만
2. 사용자 답변을 적극 인정
3. 사용자 답변을 다음 질문에 반영
4. 구체적 예시 제공
5. 충분한 시간 제공 (성급하지 않기)
```

### DON'T ❌

```
1. 연쇄 질문 (한 번에 여러 질문)
2. 제의 강요 ("해야 한다")
3. 사용자의 생각을 무시
4. 지나친 개입
5. 성급한 결론 도출
```
